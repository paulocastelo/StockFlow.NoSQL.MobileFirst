# StockFlow NoSQL / Mobile First

StockFlow NoSQL / Mobile First reuses the same inventory domain from StockFlow Core but shifts the implementation toward a mobile-first experience and a NoSQL data model.

## Goal

Show technical versatility by solving a familiar business problem with a different stack and data modeling strategy.

## Scope

- products
- stock by location
- simplified movement records
- quick lookup flows
- mobile-first usage

## Suggested Stack

- Back-end: Spring Boot
- Database: MongoDB
- Mobile: Flutter
- Optional Web: minimal admin or documentation surface

## What This Repository Should Explain

- why NoSQL was chosen
- what changed compared to the relational version
- benefits and trade-offs

## Delivery Focus

1. API designed around mobile flows
2. document-based modeling
3. Flutter experience
4. comparison notes versus StockFlow Core
5. tests and documentation

## Initial Structure

- `backend`
- `mobile`
- `docs`

## Notes

This repository should make the architectural comparison explicit, not implicit.
